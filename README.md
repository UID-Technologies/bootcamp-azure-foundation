## **Azure Foundation – Cloud Engineer**

**Platform:** Microsoft Azure
**Role:** Cloud Engineer
**Level:** Foundation → Intermediate
**Total Sessions:** 6
**Session Duration:** 2 Hours each
**Total Duration:** 16 Hours

* Theory: 4 hrs
* Hands-on: 9 hrs
* Capstone: 3 hrs

---

## **Session 1 (2 Hours): Azure Fundamentals & Resource Governance**

**Focus:** How Azure is structured and governed in enterprises

* Azure global infrastructure: regions, availability zones
* Subscriptions, management groups, and resource groups
* Azure identity basics and RBAC model
* Built-in roles vs custom roles
* Governance fundamentals:

  * Naming conventions
  * Tags for cost & ownership
  * Azure Policy overview
* Hands-on governance walkthrough:

  * Create resource groups
  * Apply tags
  * Assign RBAC roles
  * Validate policy enforcement

**Outcome:**
Understand **how Azure resources are organized, secured, and governed**.

---

## **Session 2 (2 Hours): Networking Foundations – VNet, Subnets & NSGs**

**Focus:** Secure network design for Azure workloads

* VNet architecture and address space planning
* Subnets and segmentation strategies
* Network Security Groups (NSGs): inbound & outbound rules
* Service endpoints and private connectivity (conceptual)
* VNet peering overview
* Azure Bastion and jumpbox patterns
* Hands-on networking build:

  * Create VNet with multiple subnets
  * Apply NSGs
  * Test connectivity securely

**Outcome:**
Design and validate **secure Azure virtual networks**.

---

## **Session 3 (2 Hours): Compute Fundamentals – Azure Virtual Machines**

**Focus:** VM provisioning, configuration, and availability

* Azure VM architecture and use cases
* VM sizing and image selection
* OS disks vs data disks
* Availability sets vs availability zones
* VM extensions and boot diagnostics
* cloud-init basics (Linux)
* Hands-on VM deployment:

  * Deploy Linux/Windows VM
  * Attach and format data disk
  * Enable diagnostics
  * Validate SSH/RDP access

**Outcome:**
Provision and operate **production-ready Azure VMs**.

---

## **Session 4 (2 Hours): Azure Storage Services**

**Focus:** Secure and cost-optimized data storage

* Storage account types (GPv2, Premium)
* Redundancy options: LRS, ZRS, GRS
* Blob Storage and Azure Files use cases
* Containers, file shares, and access models
* Lifecycle management policies
* Shared Access Signatures (SAS)
* Hands-on storage setup:

  * Create storage account
  * Configure lifecycle rules
  * Generate SAS
  * Mount Azure Files on VM
  * Upload and manage data

**Outcome:**
Design and manage **secure, resilient Azure storage**.

---

## **Session 5 (2 Hours): Security, Monitoring, Cost Management & IaC Intro**

**Focus:** Operability, visibility, and repeatability

* Azure Monitor: metrics and logs
* Log Analytics workspace
* Diagnostic settings for VMs & storage
* Azure Advisor overview
* Cost Management:

  * Budgets
  * Alerts
  * Cost analysis
* Security foundations:

  * Managed identities
  * Azure Key Vault overview
* Infrastructure-as-Code introduction:

  * ARM vs Bicep
  * Simple declarative deployment
* Hands-on:

  * Enable VM insights
  * Configure diagnostics
  * Create cost budget & alert
  * Deploy a basic resource using Bicep

**Outcome:**
Operate Azure workloads with **visibility, security, and cost control**.

---

## **Session 6 (2 Hours): Capstone Project – VM + VNet + Storage**

**Focus:** Integrated, hands-on implementation (no theory)

* Design an end-to-end Azure solution:

  * VNet with subnets & NSGs
  * VM deployed into secure network
  * Storage account integrated with VM
* Apply governance:

  * Tags
  * RBAC
  * Naming standards
* Enable:

  * Monitoring & diagnostics
  * Cost visibility
* Deliverables:

  * Architecture diagram
  * Deployment scripts (Bicep/ARM or CLI)
  * Validation checklist
  * Short runbook

**Outcome:**
Demonstrate **end-to-end Azure infrastructure skills** aligned to real Cloud Engineer responsibilities.
